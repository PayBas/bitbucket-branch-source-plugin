A Java regular expression to exclude specific pull request branch names from being affected by the selected naming strategy. Names that match the supplied regular expression will not be changed (their original branch names will be used).

Example: use ^release/.*|^hotfix/.* to exclude all release/* and hotfix/* branches.
Leave empty to apply the selected naming strategy to all pull requests.

NOTE: Pull request discovery strategy classifiers (merge or head) will still be appended (or prepended, depending on the naming strategy) if multiple pull request "discovery strategies" are active at the same time. Even for branches matching the specified pattern.
