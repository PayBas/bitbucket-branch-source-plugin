Determines the display name of pull requests in Jenkins.
Use pull request ID (default)
Transforms the branch name to "PR-#", example: "PR-24"
Use pull request ID and title (experimental) *
Transforms the branch name to "PR-#: {pr-title}", example: "PR-24: My feature title"
Note that whitespace characters will occur! This can potentially conflict with other Jenkins plugins.
Use pull request ID and branch name
Transforms the branch name to "PR-#_{branch-name}", example: "PR-24_feature/my-branch-name"
Use only pull request title (experimental) *
Transforms the branch name to "{pr-title}", example: "My feature title"
Note that whitespace characters will occur! This can potentially conflict with other Jenkins plugins.
Use only pull request branch name
Transforms the branch name to "{branch-name}", example: "feature/my-branch-name"

NOTE: Any selected strategy will not affect the branch or pull request names on Bitbucket.

NOTE: Pull request discovery strategy classifiers (merge or head) will still be appended (or prepended, depending on the naming strategy) if multiple pull request "discovery strategies" are active at the same time.

* Changing the title of the pull request in Bitbucket will currently cause any previous builds of said pull request to be discarded, and a new build branch to be created. It is currently not advised to use this strategy if project builds tend to be time consuming.
